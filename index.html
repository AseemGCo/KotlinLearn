CHAPTER 01
Variables
Data ko naam dena — val aur var ka fark
val — Permanent Value
Immutable  val se banaya variable change nahi ho sakta. Ek baar value assign karo, hamesha ke liye wahi rahegi.
Hindi Example
Jaise Aadhaar card number — ek baar assign hua, zindagi bhar nahi badlega. Ya date of birth — fix hai, kabhi change nahi hogi. 
val aadharNumber = "1234-5678-9012"    // ek baar set, hamesha same
val dateOfBirth  = "15-Aug-1998"
val pi           = 3.14159

aadharNumber = "9999"  // ERROR ✗ — Val cannot be reassigned
Kyu Chahiye?
Accidental changes se protect karta hai. Code mein intent clear hoti hai — "yeh value change nahi honi chahiye." Thread-safe bhi hai. Jab doubt ho, pehle val try karo.
Andar Kya Hota Hai?
val = Java ka final keyword. JVM stack ya heap pe store hoti hai, lekin reference lock ho jaata hai. Compiler compile-time pe hi error de deta hai agar reassign karo.
var — Variable Value
Mutable  var se banaya variable change ho sakta hai. Jab value update karni ho tab use karo.
Hindi Example
Jaise bank balance — aaj 5000, kal 10000, parso 0. Ya game ka score — har action ke baad update hota rehta hai. 
var bankBalance = 50000
bankBalance = 60000    // OK ✓
bankBalance += 5000   // OK ✓ — now 65000

var gameScore = 0
gameScore++              // OK ✓
Type Inference — Kotlin Khud Samajhta Hai
Kotlin compiler value dekh ke khud type samajh leta hai. Explicitly likhna optional hai.
val name    = "Rahul"      // String — compiler ne khud jaana
val age     = 25          // Int
val price  = 99.5        // Double
val active = true        // Boolean

// Explicitly type likhna — dono sahi hain:
val city: String = "Delhi"
var count: Int    = 0
Kyu Zaruri?
Java mein har baar likhna padta tha: String name = "Rahul". Kotlin mein compiler itna smart hai ki type khud deduce kar le. Code shorter, readable, aur kam error-prone.
Lazy Initialization — Baad Mein Banao
by lazy = value tabhi banao jab pehli baar use ho. Costly operations ke liye perfect.
Hindi Example
Jaise ghar mein RO water purifier — tab on hota hai jab pani nikalna ho, hamesha on nahi rehta. Resources bachao. 
val heavyData: List<String> by lazy {
    println("Abhi data load ho raha hai...")
    fetchFromDatabase()   // costly operation
}

// Yahan tak heavyData nahi bana
println("Program shuru hua")

// Ab pehli baar use kiya — tabhi banata hai
println(heavyData.size)   // "Abhi data load ho raha hai..." print hoga
Andar Kya Hota Hai?
by lazy ek delegate pattern hai. Internally ek SynchronizedLazyImpl object banta hai jo pehli access pe lambda run karta hai, result cache karta hai. Thread-safe by default.

Type Conversion — Ek type se doosre mein
Kotlin mein implicit conversion nahi hoti — explicitly convert karna padta hai.
val num: Int = 42

// Int → Double, String, Long etc.
val d: Double = num.toDouble()
val s: String = num.toString()
val l: Long   = num.toLong()

// String → Int (careful — crash if invalid!)

val safe = "abc".toIntOrNull() // null — crash nahi hoga

// Java jaisi automatic conversion NAHI hoti:
val x: Int    = 5

}
CHAPTER 03
Functions
Kaam ka dabba — baar baar use karo
Basic Function
Hindi Example
Chai banane ki recipe — ek baar likhi, jab bhi chahiye call karo. Input: cups, paani. Output: ready chai. 
// fun keyword — functionName(params): ReturnType
fun chaiBanao(cups: Int, sugar: Boolean = true): String {
    val sweetness = if (sugar) "meethi" else "phiki"
    return "$cups cup $sweetness chai ready!"
}

chaiBanao(2)              // "2 cup meethi chai ready!"
chaiBanao(1, false)       // "1 cup phiki chai ready!"
Expression Body — Single Line Functions
Jab function sirf ek value return kare — curly braces aur return hatao.
// Verbose style:
fun double(n: Int): Int {
    return n * 2
}

// Expression body — same kaam, 1 line:
fun double(n: Int) = n * 2

fun isAdult(age: Int) = age >= 18
fun fullName(first: String, last: String) = "$first $last"
fun max(a: Int, b: Int) = if (a > b) a else b
Default Parameters + Named Arguments
Hindi Example
Pizza order karte waqt — "medium size, cheese extra, no olives." Har baar sab nahi bolna padta — jo default chahiye woh assume ho jaata hai. 
fun pizzaOrder(
    size   : String  = "medium",
    cheese : Boolean = true,
    olives : Boolean = false,
    qty    : Int     = 1
) {
    println("$qty x $size pizza, cheese=$cheese, olives=$olives")
}

pizzaOrder()                             // sab default
pizzaOrder("large")                      // sirf size change
pizzaOrder(qty = 3, olives = true)    // named — order matter nahi
pizzaOrder(size = "small", cheese = false) // ekdum clear
Kyu Important?
Java mein default values ke liye multiple overloaded methods likhne padte the. Kotlin mein ek function, sab handle. Named arguments se 5-6 parameter wale functions bhi readable rehte hain.
Extension Functions — Purani class mein naye kaam
Kisi bhi class mein — apni ya library ki — naya function add kar sakte ho bina usse inherit kiye.
// String class mein naya function add kar rahe hain
fun String.isPanCard(): Boolean {
    return this.matches("[A-Z]{5}[0-9]{4}[A-Z]{1}".toRegex())
}

fun Int.rupees() = "₹$this"

// Ab String objects pe seedha call kar sakte ho:
println("ABCDE1234F".isPanCard())  // true
println("HELLO".isPanCard())       // false
println(5000.rupees())            // ₹5000
Andar Kya Hota Hai?
Extension functions bytecode mein static methods ban jaati hain. this = receiver object. Class ka structure nahi badalta — sirf ek illusion hai. Android SDK mein yeh bahut use hoti hain.
vararg — Kitne bhi arguments
fun totalMarks(vararg marks: Int): Int {
    return marks.sum()
}

totalMarks(80, 90)                // 170
totalMarks(70, 85, 92, 78)       // 325

// Array already hai? Spread operator * use karo:
val scores = intArrayOf(85, 90, 95)
totalMarks(*scores)               // 270
CHAPTER 04
Null Safety
NullPointerException — Kotlin ka sabse bada gift
Problem — Null kya hai?
Hindi Example
Socho tumne kisi ka naam variable mein store kiya, lekin unhone bataya hi nahi. Woh value "null" hai — khali haath. Jab tum us khali haath se kuch kaam karne ki koshish karo — crash! 
// Java mein yeh valid tha — aur yahi problem thi:
String name = null;
name.length();   // NullPointerException — app crash

// Kotlin mein default mein null allowed hi nahi:
var name: String = null   // ERROR — compile time pe hi!
Kyu Important?
Billion-dollar mistake — Tony Hoare (null ke inventor) ne khud kaha yeh unki sabse badi galti thi. Android apps mein NPE sabse common crash reason tha. Kotlin ne compile-time pe hi band kar diya.
Nullable Types — ? lagao
// Non-nullable — value JARUR hogi:
var name: String  = "Rahul"
name = null              // ERROR ✗

// Nullable — ? lagao, null ho sakta hai:
var nickname: String? = null
nickname = "Rocky"        // OK ✓
nickname = null           // OK ✓

// Common nullable cases:
var middleName: String? = null   // not everyone has it
var profilePic: String? = null   // might not be set
var lastLogin : Long?   = null   // new user hasn't logged in
Safe Call Operator — ?.
Nullable variable pe safely kaam karo. Agar null hai → null return karo, crash nahi.
val nickname: String? = null

// Bina safe call — crash:
println(nickname.length)    // ERROR — compiler rok dega

// Safe call — null hai toh null return karo:
println(nickname?.length)   // prints: null (no crash)

// Chain karo:
val user: User? = getUser()
val city = user?.address?.city  // null agar koi bhi null ho
Elvis Operator — ?:
Hindi Example
"Agar naam hai toh woh dikhao, nahi hai toh 'Guest' dikhao." — Yahi Elvis operator karta hai. Null ho toh fallback value do. 
val nickname: String? = null

// Long way:
val display = if (nickname != null) nickname else "Guest"

// Elvis way — same kaam:
val display = nickname ?: "Guest"

// Return ya throw bhi kar sakte ho:
fun getUser(id: Int): User {
    return findUser(id) ?: throw Exception("User $id not found")
}
let + null check — Most Common Pattern
val email: String? = getUserEmail()

// Sirf tab run karo jab email null nahi:
email?.let { e ->
    sendWelcomeEmail(e)
    println("Email bheja: $e")
}

// if-else ke saath:
email?.let {
    println("Email: $it")
} ?: println("Email nahi diya")
!! Operator — Danger Zone
Warning
!! matlab: "Main guarantee de raha hoon yeh null nahi hai." Agar null nikla — NPE crash. Use karo sirf jab 100% sure ho. 
val name: String? = getName()

// Risky — crash if null:
println(name!!.length)

// Prefer safe patterns:
println(name?.length ?: 0)  // better
CHAPTER 05
Classes
Blueprint banana — Objects ka khaandaan
Basic Class
Hindi Example
Class = ghar ka naksha. Naksha ek hi, ghar hazaar bana sakte ho. Har ghar ka apna address (properties) aur apna kaam (functions). 
class Student(val name: String, var marks: Int) {

    // Property with custom getter
    val grade: String
        get() = when {
            marks >= 90 -> "A"
            marks >= 75 -> "B"
            marks >= 60 -> "C"
            else        -> "F"
        }

    fun isPass() = marks >= 33

    fun printReport() {
        println("$name | Marks: $marks | Grade: $grade | ${if (isPass()) "Pass" else "Fail"}")
    }
}

// Object banao — new keyword nahi!
val s1 = Student("Anil", 85)
val s2 = Student("Neha", 25)

s1.printReport()  // Anil | Marks: 85 | Grade: B | Pass
s2.printReport()  // Neha | Marks: 25 | Grade: F | Fail

// Marks update karo:
s2.marks = 45
s2.printReport()  // Neha | Marks: 45 | Grade: C | Pass
Andar Kya Hota Hai?
Primary constructor JVM <init> ban jaata hai. val = private field + getter. var = private field + getter + setter. Kotlin yeh sab automatically generate karta hai.
Data Class — Sirf Data Rakhne Ke Liye
data class User(val id: Int, val name: String, val city: String)

val u1 = User(1, "Meera", "Pune")
val u2 = u1.copy(city = "Mumbai")   // ek cheez change, baaki same

println(u1)             // User(id=1, name=Meera, city=Pune)
println(u1 == u2)       // false — content compare hota hai

// Destructuring:
val (id, name, city) = u1
println("$name lives in $city")  // Meera lives in Pune
Auto-Generated Methods:
toString() — readable print  •  equals() — content comparison  •  hashCode() — HashMap ke liye  •  copy() — modified copy  •  componentN() — destructuring. Java mein yeh sab manually likhna padta.
Sealed Class — Limited Khaandaan
Hindi Example
API response sirf teen ho sakta hai: Success, Loading, ya Error. Sealed class kehti hai: "Is class ki subclasses sirf yahi hain, aur koi nahi." when se sab handle karo — compiler ensure karta hai koi case miss na ho. 
sealed class ApiResult<T> {
    data class  Success<T>(val data: T)      : ApiResult<T>()
    data class  Error(val message: String)   : ApiResult<Nothing>()
    object      Loading                     : ApiResult<Nothing>()
}

fun handleResult(result: ApiResult<User>) {
    when (result) {
        is ApiResult.Success  -> showUser(result.data)
        is ApiResult.Error    -> showError(result.message)
        ApiResult.Loading     -> showLoader()
    }  // else ki zarurat nahi — compiler jaanta hai sab cases cover hain
}
Inheritance + Open Classes
// Kotlin mein default final hai — explicitly open karo
open class Vehicle(val brand: String) {
    open fun describe() = println("$brand vehicle")
}

class Car(brand: String, val doors: Int) : Vehicle(brand) {
    override fun describe() = println("$brand car with $doors doors")
}

class Bike(brand: String) : Vehicle(brand)

val c = Car("Maruti", 4)
c.describe()   // Maruti car with 4 doors
Kyu Default Final?
Effective Java ki guidance: "Design for inheritance or prohibit it." Accidentally extend karna bugs ka source hai. Kotlin mein explicitly open likhna = conscious decision.
Object + Companion Object — Singleton Pattern
// object = singleton — sirf ek instance
object AppConfig {
    val BASE_URL = "https://api.example.com"
    val TIMEOUT  = 30
}

println(AppConfig.BASE_URL)  // new nahi karna padta

// companion object = Java's static members
class User(val name: String) {
    companion object {
        fun create(name: String) = User(name)
        const val MAX_NAME_LENGTH = 50
    }
}

val u = User.create("Priya")  // factory method

Set — Unique Values Only
Hindi Example
Lucky draw ke liye ek box mein unique slips. Duplicate slip daalo — woh accept hi nahi hogi. Set same kaam karta hai — duplicates ignore hote hain. 
val uniqueMarks = setOf(90, 85, 90, 75, 85)
// Result: {90, 85, 75} — duplicates remove

Map — Key → Value
Hindi Example
Phone book — naam (key) → number (value). Naam se number dhundo. Har naam unique hoga, lekin ek number do logon ke paas ho sakta hai. 
val phoneBook = mapOf(
    "Rahul" to "9876543210",
    "Priya" to "8765432109",
    "Amit"  to "7654321098"
)

phoneBook["Rahul"]           // "9876543210"
phoneBook["Kiran"]           // null — nahi mila
phoneBook.getOrDefault("Kiran", "Unknown")  // "Unknown"
phoneBook.containsKey("Priya")   // true
phoneBook.keys     // [Rahul, Priya, Amit]
phoneBook.values   // [9876543210, ...]

// Map pe loop:
for ((name, number) in phoneBook) {
    println("$name: $number")
}
Collection Functions — Power Tools
Yeh Kotlin ka asli jadoo hai. filter, map, reduce, groupBy — sab ek line mein.
val marks = listOf(45, 82, 29, 91, 33, 17, 76, 58)

// filter — condition wale rakho
val passed   = marks.filter { it >= 33 }    // [45,82,91,33,76,58]

// map — transform each element
val percent  = marks.map { it * 100 / 100 }

// sortedBy, sortedDescending
val topFirst = marks.sortedDescending()

// any / all / none
marks.any  { it > 90 }   // true — koi ek 90+ hai?
marks.all  { it > 30 }   // false — sab 30+ nahi
marks.none { it > 100}   // true — koi 100+ nahi

// count, sum, average, max, min
marks.count { it >= 60 }  // kitne 60+?
marks.average()            // average

// groupBy — category mein sort karo
val byResult = marks.groupBy { if (it >= 33) "Pass" else "Fail" }
// {Pass=[45,82,91,33,76,58], Fail=[29,17]}

// Chain karo:
val topStudents = marks
    .filter { it >= 60 }
    .sortedDescending()
    .take(3)       // top 3
Performance Tip
Har step naya list banata hai. Bahut badi list (10K+ items) ke liye asSequence() use karo — lazy evaluation, ek hi pass:

marks.asSequence().filter { it > 60 }.map { it * 2 }.toList()

Control Flow
Faisla karna aur ghoomna phirna
if — Expression Bhi Hai, Statement Bhi
Kotlin mein if ek value return kar sakta hai — ternary operator ki zarurat nahi.
val score = 78

// Statement style:
if (score >= 33) println("Pass") else println("Fail")

// Expression style — value return karta hai:
val result = if (score >= 33) "Pass" else "Fail"

// Multi-line expression:
val grade = if (score >= 90)      "A — Topper!"
             else if (score >= 75) "B — First Class"
             else if (score >= 60) "C — Second Class"
             else if (score >= 33) "D — Pass"
             else                   "F — Fail"
when — Switch Ka Baap
Hindi Example
Aacha, railway station pe announcement: "Train number X platform Y pe aayegi." Har train number ke liye alag case — yahi when hai, lekin Java ke switch se bahut zyada powerful. 
val day = 3

// Basic when:
val dayName = when (day) {
    1       -> "Monday"
    2       -> "Tuesday"
    3       -> "Wednesday"
    6, 7    -> "Weekend!"   // multiple values
    else    -> "Unknown"
}

// when with range:
val category = when (score) {
    in 90..100 -> "Distinction"
    in 75..89  -> "First Class"
    in 33..74  -> "Pass"
    else       -> "Fail"
}

// when without argument — if-else chain replace:
when {
    score > 95 -> println("State topper!")
    score > 75 -> println("Bohot acha!")
    else        -> println("Aur mehnat karo")
}

// when with type check:
fun describe(obj: Any) = when (obj) {
    is Int    -> "Number: $obj"
    is String -> "Text: ${obj.uppercase()}"
    is List*  -> "List of ${obj.size} items"
    else       -> "Unknown"
}
for Loops — Ghoomna Phirna
// Range loops:
for (i in 1..5)          print("$i ")  // 1 2 3 4 5
for (i in 1 until 5)    print("$i ")  // 1 2 3 4
for (i in 5 downTo 1)  print("$i ")  // 5 4 3 2 1
for (i in 0..10 step 2) print("$i ")  // 0 2 4 6 8 10

// List pe loop:
val states = listOf("UP", "MP", "GJ", "MH")
for (state in states) println(state)

// Index ke saath:
for ((index, state) in states.withIndex()) {
    println("$index: $state")
}

// Map pe loop:
for ((key, value) in phoneBook) {
    println("$key → $value")
}
while + do-while + break + continue
// while:
var attempts = 0
while (attempts < 3) {
    println("Attempt ${attempts + 1}")
    attempts++
}

// do-while — pehle karo, phir check:
do {
    print("Enter PIN: ")
    val pin = readLine()
} while (pin != "1234")

// break + continue:
for (i in 1..10) {
    if (i == 5) continue   // 5 skip karo
    if (i == 8) break      // 8 pe band karo

}
CHAPTER 08
Lambdas & Higher-Order Functions
Functions as values — functional programming ka darwaza
Lambda — Naamless Function
Hindi Example
Ghar pe ek thela-wala aata hai, sirf ek kaam karne ke liye — sabzi chhaanna. Uska naam nahi poochte — seedha kaam batao. Lambda waisa hi hai — naam nahi, sirf kaam. 
// Lambda syntax: { parameters -> body }
val greet  = { println("Namaste!") }
val add    = { a: Int, b: Int -> a + b }
val isEven = { n: Int -> n % 2 == 0 }

greet()            // Namaste!
println(add(3, 4)) // 7
println(isEven(4)) // true

// Lambda type explicitly:
val multiply: (Int, Int) -> Int = { a, b -> a * b }
it — Single Parameter Shortcut
val numbers = listOf(1, 2, 3, 4, 5)

// Parameter naam dena — verbose:
numbers.filter { num -> num > 2 }

// it use karo — same kaam:
numbers.filter { it > 2 }         // [3, 4, 5]
numbers.map    { it * 2 }         // [2, 4, 6, 8, 10]
numbers.filter { it % 2 == 0 }   // [2, 4] — even numbers
Higher-Order Functions — Lambda Pass Karo
Hindi Example
Socho ek "courier service" hai. Tum bolo: "Yeh item le jaao, yeh kaam karo." Kya kaam karna hai — tum batate ho, courier service execute karta hai. Lambda = "yeh kaam" hai jo tum dete ho. 
// Function jo dusra function accept kare:
fun calculate(a: Int, b: Int, operation: (Int, Int) -> Int): Int {
    return operation(a, b)
}

calculate(10, 5) { a, b -> a + b }   // 15
calculate(10, 5) { a, b -> a * b }   // 50
calculate(10, 5) { a, b -> a - b }   // 5

// Real world — button click listener (Android):
button.setOnClickListener {
    println("Button dabaya!")
}
Trailing Lambda + Inline Functions
// Agar last parameter lambda hai — baahar nikaal sakte ho:
fun doTask(times: Int, task: () -> Unit) {
    repeat(times) { task() }
}

// Andar lambda:
doTask(3, { println("Kaam karo") })

// Trailing lambda style — zyada readable:
doTask(3) { println("Kaam karo") }

// inline function — lambda code calling site pe paste hoti hai
@inline
fun measure(block: () -> Unit): Long {
    val start = System.currentTimeMillis()
    block()
    return System.currentTimeMillis() - start
}
Andar Kya Hota Hai?
Normal lambda = anonymous class object. inline = lambda body literally calling site pe copy ho jaata hai — zero object overhead. Performance critical code mein use karo.

let — Null Check + Transform
Hindi Example
"Agar parcel aaya hai (not null), toh andar aane do aur kaam karo." let sirf tab chalega jab value null nahi hogi. 
// Null safe execution:
val email: String? = getEmail()

email?.let {
    sendEmail(it)          // it = email value
    println("Sent to $it")
}

// Transform:
val length = email?.let { it.length } ?: 0

// Scope limit karna (temp variable se bachao):
val result = someComplexCalculation().let {
    if (it > 100) "High" else "Low"
}
apply — Object Ko Configure Karo
Hindi Example

Problem — Threads Se Kya Problem Tha?
Hindi Example
Dukaan ka owner (main thread) har cheez khud kare — parcel bhi uthaye, register bhi bhare, customer bhi sambhale. Jab tak parcel wala aata hai — sab kaam ruk gaya. Thread block = yahi hota hai. Coroutines = helper hire karo, owner apna kaam karta rahe. 
// Thread block karta hai — BAD (Android mein app freeze hogi):
fun fetchData() {
    Thread.sleep(3000)  // 3 sec ke liye UI freeze
    println("Data aaya")
}

// Coroutine — thread block nahi karta:
suspend fun fetchData() {
    delay(3000)        // suspend karta hai — thread free hai
    println("Data aaya")
}
Thread vs Coroutine
Thread = OS resource, ~1MB memory. Coroutine = Kotlin runtime ka concept, sirf kuch bytes. Ek thread pe hazaron coroutines chal sakti hain. Context switch bhi cheap hai.
suspend Function — Ruk sakti hai, block nahi karti
// suspend = yeh function pause ho sakta hai
suspend fun getUserFromApi(id: Int): User {
    delay(1000)                    // network call simulate
    return User(id, "Ravi", "Delhi")
}

// suspend function sirf:
// 1. Dusri suspend function se call ho sakti hai
// 2. Ya coroutine builder se (launch, async etc.)

// Normal function se direct nahi:
fun normalFun() {
    getUserFromApi(1)  // ERROR — suspend function yahan nahi chalegi
}
Coroutine Builders — launch, async, runBlocking
import kotlinx.coroutines.*

// runBlocking — testing mein use karo, production mein nahi
fun main() = runBlocking {

    // launch — fire and forget (no return value)
    val job = launch {
        delay(1000)
        println("Background kaam ho gaya")
    }

    println("Yeh pehle print hoga")
    job.join()    // wait karo finish hone ka

    // async — result chahiye
    val deferred = async {
        delay(500)
        42    // return value
    }
    val result = deferred.await()   // block until result
    println("Result: $result")        // 42
}
Parallel Execution — Dono Kaam Ek Saath
Hindi Example
Ek kaam — pehle dal chadhao, phir sabzi kaato. Sequential = 20 min. Smart = dono ek saath — sirf 12 min. async se parallel chalao. 
suspend fun getTotal(): Int = coroutineScope {

    // Sequential — 2 sec total:
    val price1 = fetchPrice1()   // 1 sec wait
    val price2 = fetchPrice2()   // 1 sec wait

    // Parallel — 1 sec total!
    val p1 = async { fetchPrice1() }   // starts immediately
    val p2 = async { fetchPrice2() }   // starts immediately
    p1.await() + p2.await()   // dono ka intezaar
}

Exception Handling + Cancellation
// try-catch — coroutines mein bhi normal kaam karta hai:
launch {
    try {
        val data = fetchData()
        processData(data)
    } catch (e: Exception) {
        println("Error: ${e.message}")
    }
}

// CoroutineExceptionHandler:
val handler = CoroutineExceptionHandler { _, e ->
    println("Caught: $e")
}

launch(handler) {
    throw RuntimeException("Kuch toot gaya")
}

// Cancellation:
val job = launch { longRunningTask() }
job.cancel()   // cancel karo
Android Tip

Andar Kya Hota Hai?
suspend function compiler state machine mein convert karta hai. Har suspend point ek state ban jaata hai. JVM stack frame save hoti hai, thread release hoti hai. Jab continue karna ho — state machine resume hoti hai. Yahi coroutines ka jadoo hai.
